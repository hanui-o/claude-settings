# Claude Code 확장 기능 이해하기

Claude Code는 기본 코딩 기능 외에 다양한 확장 기능을 제공합니다. 이 문서에서는 각 기능의 역할과 언제 어떻게 조합해서 사용하는지 설명합니다.

> 관련 문서: [시작 가이드](./getting-started.md) · [메인 README](../README.md)
>
> 공식 문서: [Claude Code 확장하기](https://code.claude.com/docs/ko/features-overview)

---

## 확장 기능 한눈에 보기

| 기능 | 하는 일 | 언제 쓰나 | 예시 |
|------|---------|-----------|------|
| **CLAUDE.md** | 모든 세션에서 자동 로드되는 프로젝트 규칙 | "항상 이렇게 해" 규칙 | "pnpm 사용, 커밋 전 테스트 실행" |
| **Skill (커맨드)** | 온디맨드 지식·워크플로우 | 반복 작업, 참조 문서 | `/deploy`, `/review` |
| **Subagent (에이전트)** | 독립 컨텍스트에서 작업 후 결과만 반환 | 많은 파일 분석, 병렬 작업 | code-reviewer, test-runner |
| **MCP** | 외부 서비스 연결 | DB, GitHub, Slack 등 접근 | DB 쿼리, Slack 메시지 전송 |
| **Hook** | 이벤트 발생 시 자동 실행되는 스크립트 | 자동화, 검증, 안전장치 | 파일 저장 후 Prettier, 위험 명령 차단 |
| **Plugin** | 위 기능들을 묶어서 배포하는 패키지 | 팀/커뮤니티 공유 | 여러 skill + hook을 하나로 번들 |

## CLAUDE.md vs Skill (커맨드)

둘 다 Claude에게 지침을 주지만, **로드 타이밍**이 다릅니다.

| | CLAUDE.md | Skill (커맨드) |
|---|---|---|
| **로드** | 모든 세션에서 자동 | 호출할 때만 (`/명령어`) |
| **용도** | "항상 이렇게 해" 규칙 | 특정 작업의 워크플로우, 참조 문서 |
| **크기** | 500줄 이하 권장 | 제한 없음 (필요할 때만 로드) |
| **예시** | "pnpm 사용, main 직접 커밋 금지" | `/deploy` 배포 절차, `/review` 리뷰 체크리스트 |

**경험 법칙:** CLAUDE.md가 길어지면 참조 내용을 Skill로 분리하세요.

---

## Skill vs Subagent (에이전트)

| | Skill (커맨드) | Subagent (에이전트) |
|---|---|---|
| **실행 컨텍스트** | 현재 대화 안에서 실행 | 독립 컨텍스트에서 실행, 요약만 반환 |
| **장점** | 대화 흐름을 유지 | 메인 대화가 깔끔하게 유지됨 |
| **적합한 작업** | 참조 문서, 단순 워크플로우 | 많은 파일을 읽는 분석, 병렬 작업 |
| **예시** | `/commit` → 커밋 규칙대로 실행 | code-reviewer → 수십 개 파일 분석 후 리포트 |

**둘을 조합할 수도 있습니다.** Subagent가 특정 Skill을 미리 로드해서 사용할 수 있습니다.

## MCP vs Skill

서로 다른 문제를 해결하며, 함께 쓸 때 가장 강력합니다.

| | MCP | Skill |
|---|---|---|
| **제공하는 것** | 외부 서비스 **접근 능력** | 그 서비스를 **잘 사용하는 방법** |
| **예시** | DB에 연결 | DB 스키마와 쿼리 패턴 문서화 |

**조합 예시:** MCP로 PostgreSQL에 연결하고, Skill에 "주문 테이블은 `orders`, 사용자 조회는 항상 `user_id` 인덱스를 타야 함" 같은 지식을 담음.

## 컨텍스트 비용

추가하는 모든 기능은 Claude의 컨텍스트 윈도우를 소비합니다. 너무 많으면 성능이 떨어집니다.

| 기능 | 로드 시점 | 컨텍스트 비용 |
|------|-----------|---------------|
| **CLAUDE.md** | 세션 시작 | 매 요청마다 (항상 포함) |
| **Skill** | 호출 시 | 호출 전엔 설명만 (가벼움) |
| **Subagent** | 생성 시 | 메인 세션과 격리 (영향 없음) |
| **MCP** | 세션 시작 | 도구 정의만큼 (서버 많으면 커짐) |
| **Hook** | 이벤트 시 | 0 (외부에서 실행) |

### 가볍게 유지하는 팁

- CLAUDE.md는 **500줄 이하**, 길어지면 Skill로 분리
- 자주 안 쓰는 MCP 서버는 연결 해제 (`claude mcp remove`)
- `/mcp` 명령으로 서버당 토큰 비용 확인 가능

## 실전 조합 패턴

### Skill + MCP: 외부 서비스를 똑똑하게

```
MCP → DB 연결
Skill → DB 스키마, 쿼리 패턴, 주의사항 문서화
```

Claude가 DB에 접근할 수 있을 뿐 아니라, 어떻게 쿼리해야 하는지도 알게 됩니다.

### Skill + Subagent: 병렬 리뷰

```
/review Skill → 보안, 성능, 스타일 Subagent 각각 생성
각 Subagent → 독립 컨텍스트에서 분석 후 결과 반환
```

하나의 커맨드로 여러 관점의 리뷰를 동시에 실행합니다.

### CLAUDE.md + Skill: 규칙과 참조 분리

```
CLAUDE.md → "API 규칙을 따르세요" (한 줄)
Skill → 전체 API 스타일 가이드 (100줄+)
```

핵심 규칙은 항상 로드하고, 상세 참조는 필요할 때만 로드합니다.

### Hook + MCP: 자동 알림

```
Hook → 중요 파일 수정 감지 (PostToolUse)
MCP → Slack에 자동 알림 전송
```

## 이 레포에서 제공하는 것

| Claude Code 기능 | 이 레포의 파일 | 설치 위치 |
|-------------------|----------------|-----------|
| Skill (커맨드) | `commands/*.md` | `.claude/commands/` |
| Subagent (에이전트) | `agents/*.md` | `.claude/agents/` |
| Hook | `hooks/*.sh` | `.claude/hooks/` |
| CLAUDE.md | `CLAUDE.md.template` | 프로젝트 루트 `CLAUDE.md` |

MCP와 Plugin은 프로젝트마다 다르므로 이 레포에 포함하지 않습니다. MCP 설정 방법은 [시작 가이드](./getting-started.md#3-claude-mcp-연결하기)를 참고하세요.
