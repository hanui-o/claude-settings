DB 시드 데이터를 생성하거나 실행해줘.

## 인자

- `$ARGUMENTS`: 대상 테이블 또는 "run" (기본값: 변경된 모델 기반)
  - 테이블명: 해당 테이블의 시드 파일 생성
  - `run`: 기존 시드 파일 실행
  - `reset`: 시드 데이터 초기화 후 재실행

## 절차

1. 프로젝트의 시드 도구를 감지한다
   - Prisma: `prisma/seed.ts`
   - TypeORM: `src/seeds/`
   - Knex: `seeds/`
   - Sequelize: `seeders/`
   - 직접 스크립트 방식

2. 기존 모델/스키마를 읽고 시드 데이터를 생성한다
   - 필드 타입에 맞는 현실적인 한국어 데이터 생성
   - 이름: 김철수, 이영희, 박지민 등
   - 이메일: user1@example.com 형식
   - 전화번호: 010-1234-5678 형식
   - 주소: 서울특별시 강남구 등
   - 관계 데이터 간 참조 무결성 보장
   - 최소 10건 이상의 데이터

3. `run` 또는 `reset`인 경우 시드를 실행한다
   - `reset`: 기존 데이터 삭제 후 재실행 (확인 요청)

4. 결과를 요약한다 (테이블별 생성 건수)

## 주의

- 비밀번호는 해시 처리한다 (bcrypt 등)
- 외래 키 순서를 고려하여 시드 순서를 정한다
- 프로덕션 환경에서는 실행하지 않는다
